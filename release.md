# How To Write Releases

**GitHub Releases**は、リポジトリ内のソフトウェアやプロジェクトのバージョンを管理し、特定のリリースに関連するファイル（バイナリやドキュメント）を提供する機能です。
ユーザーは過去のバージョンや更新履歴を簡単に確認できるため、アプリケーションの配布や管理に便利です。

## GitHub Releases の使い方

### 1. リリースの作成手順

1. **GitHub でリポジトリを開く**
   自分のプロジェクトのリポジトリにアクセスします。

2. **"Releases" タブに移動**
   リポジトリ画面で、「Releases」→「Create a new release」をクリックします。

3. **バージョンを設定**

   - `Tag version`: リリースのタグ（例: `v1.0.0`）。
     - タグがまだ存在しない場合、自動的に作成されます。
   - `Target`: タグが付与されるブランチやコミットを指定（通常は `main` や `master`）。

4. **タイトルと説明を入力**

   - `Release title`: リリースの名前（例: `Version 1.0.0`）。
   - `Description`: 新機能や修正内容の説明を記入（Markdown が使用可能）。

5. **ファイルのアップロード**（オプション）
   必要であれば、ビルド済みのバイナリや関連ファイルを添付します。

6. **ドラフト保存または公開**
   「Publish release」をクリックすると、リリースが公開されます。

## リリースノートのテンプレート

リリースノートの書き方は一貫性が重要です。以下に、使いやすい Markdown 形式のテンプレートを紹介します。

```markdown
# リリースノート: v1.0.0

## 🆕 新機能

- ユーザー認証機能を追加
- プロファイル編集画面の UI を改善

## 🛠️ 修正・改善

- ログイン画面でのバグを修正 (#45)
- レスポンシブ対応を強化し、モバイルでの表示崩れを解消

## ⬆️ 更新内容

- ライブラリのアップデート:
  - React 18.2 → 18.3
  - Axios 0.27 → 0.28

## 🚨 既知の問題

- ダークモードで一部アイコンが表示されない
- API のレスポンスが遅延する場合がある

## 📦 ダウンロード

以下のバイナリファイルを使用してアプリをインストールしてください。

- [Windows 版](./project-v1.0.0.exe)
- [Mac 版](./project-v1.0.0.dmg)

### 🗓️ リリース日

2024-10-15
```

### 各セクションの説明

- **新機能 (New Features)**: 追加された新しい機能をリスト化します。
- **修正・改善 (Fixes & Improvements)**: バグ修正や既存機能の改善点を記載します。
- **更新内容 (Updates)**: 使用ライブラリやツールのアップデート情報を含めます。
- **既知の問題 (Known Issues)**: ユーザーが気を付けるべき未解決の問題をリストアップします。
- **ダウンロードリンク**: 必要に応じてバイナリやインストールファイルのリンクを提供します。

このテンプレートを使えば、リリースの内容がわかりやすく整理され、プロジェクトの管理もしやすくなります。
さらに、Markdown 形式のため、GitHub 上で見やすく表示されます。
